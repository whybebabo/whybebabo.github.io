#include <stdio.h>

#define MARCO_INT 1
const int gconst_int = 3;

int main()
{
	const int const_int = 2;		// "伪"常量，可以取址改变变量
	int i1 = MARCO_INT + 10;		// 1+10=11
	int i2 = const_int + 2;			// 2+2=4
	int i3 = gconst_int + 3;		// 3+3=6
	// int* gp = const_cast<int *>(&gconst_int);	// 会导致段错误，不能改变rodata处的值
	int* p = const_cast<int *>(&const_int);		//强制转换获取常量的地址
	*p = 30;						//改变常量的值
	int i4 = const_int + 30;		// 2+30=32
	printf("i1:%d, i2:%d, *p:%d, i3:%d, i4:%d\n",
				i1, i2, *p, i3, i4);	// i1:11, i2:4, *p:30, i3:6, i4:32
	return 0;
}

以上定义了一个宏MARCO_INT，一个全局常量gconst_int和一个局部常量const_int，可以发现第15行处，虽然已经通过强制转换改变了局部变量const_int的值为30，但i4的赋值结果仍然为34，而非60。下面通过反汇编得到的代码，可以看到编译器对全局变量、局部变量、宏的处理。

  1. 从第7~9行与第27行，可以看到编译器对全局常量和局部常量的不同处理，全局常量位于rodata(readonlydata)区域，试图以任意方式修改rodata都会导致异常，而局部常量则不在该区域，可以通过强制转换来获得指针后进行修改；
  2. 从28到30行，可以看到，编译器对宏、全局常量、局部常量的处理相同，在链接前，已经求得表达式的结果并得到立即数，在运行是直接赋给相应的变量，因此会出现i4=32而非60的情况。

2. 安全的const_cast

应该注意的是：上述中对于const常量进行const_cast操作是危险的。以下引用自C++03标准：

Except that any class member declared mutable (7.1.1) can be modified, any attempt to modify a const object during its lifetime (3.8) results in undefined behavior

简而言之，在const对象生命期内，任意试图修改改对象将导致未定义的行为。上述的const_int只不过是刚好成功了而已。

只有一种情形是保证安全的，就是将一个原本非const，而被修饰为cosnt的对象使用const_cast转换。即下面这种情况：

#include <stdio.h>

void func(const int* cp)
{
	int* p = const_cast<int *>(cp);
	*p = 100;
}

int main()
{
	int i = 0;
	func(&i);
	printf("%d", i); // 100
	return 0;
}
